# خريطة متجر HJY STORE 🚀

هذا الدليل يشرح لك أين تضع ملفاتك وكيفية تسميتها ليعمل الموقع تلقائياً وبأفضل أداء.

## 📂 هيكلية المجلدات (أين تضع ملفاتك؟)

| المجلد | نوع الملفات المدعومة | الشرح |
| :--- | :--- | :--- |
| `data/` | `.csv` | **قلب المتجر**: ضع هنا ملفات المنتجات، الأقسام (`categories.csv`)، والقوائم (`fav.csv`, `out_pro.csv` ..إلخ). |
| `photo/` | `.webp`, `.png`, `.jpg` | **صور المنتجات**: يمكنك تنظيمها في مجلدات فرعية. الموقع يبحث فيها بالاسم تلقائياً. |
| `about/` | `.txt` | **أزرار أسفل الصفحة**: اسم الملف النصي يصبح عنوان الزر، ومحتواه يظهر عند الضغط. |
| `object/` | `.txt` | **الأسئلة الشائعة**: تظهر في قسم "الأسئلة الشائعة وتفاصيل كاملة". |
| `root/` | أي نوع | **هوية الموقع**: ضع هنا الشعار (`logo.png` أو `logo.webp`) وصورة المطور (`profile.webp`). |

---

## 🏷️ قواعد التسمية (Naming Rules)

1.  **الصور**: يفضل تسمية الصورة بنفس "كود المنتج" (مثال: كود المنتج `VM01` -> اسم الصورة `VM01.webp`).
2.  **دعم الصيغ**: الموقع يدعم `webp` و `png` و `jpg` بشكل كامل.
3.  **النصوص**: أي نص داخل ملفات `.txt` تضعه بين نجمتين (مثال: `*نص عريض*`) سيظهر بخط كبير وعريض (Bold).

---

## ☁️ شرح GitHub: "Commit directly to the main branch"

عندما ترفع ملفاتك إلى GitHub، ستشاهد الخيار المشار إليه في الصورة:
-   **ماذا يعني؟**: يعني "اعتماد التغييرات فوراً في الفرع الأساسي".
-   **متى تستخدمه؟**: دائماً. بما أن موقعك يعمل من الفرع الأساسي (`main`)؛ فإن هذا الخيار هو أسرع طريقة لتحديث بيانات المنتجات والصور لتظهر للزبائن فوراً.

---

## ⚡ التعامل مع الإنترنت البطيء

الموقع مجهز برمجياً للتعامل مع ضعف الشبكة في سوريا أو المناطق ذات الإنترنت المحدود:
-   **إعادة المحاولة (Auto-Retry)**: يحاول الموقع تحميل البيانات 3 مرات تلقائياً في حال فشل الاتصال.
-   **تخزين مؤقت (Cache)**: يحفظ الموقع نسخة من البيانات في جهاز الزبون ليفتح "فوراً" في المرات القادمة حتى لو كان الإنترنت مقطوعاً أو ضعيفاً.
-   **تحميل ذكي**: يتم تحميل الهيكل الأساسي للموقع أولاً، ثم البيانات، ثم الصور تدريجياً لضمان عدم تجمّد الصفحة.

---

## 🔄 كيف تحديث الموقع؟
1. أضف ملفاتك الجديدة (صور أو CSV) في مجلداتها.
2. إذا كنت تعمل محلياً، شغّل الأمر: `python scripts/generate_manifest.py`.
3. ارفع الملفات إلى GitHub واختر **Commit directly to the main branch**.
